# Task DAG —— 依赖关系驱动的任务管理工具 设计文档

日期：2026-06-11
状态：已确认

## 概述

一个自包含的单 HTML 页面工具，以流程图/思维导图的形式管理项目任务。
用户将一个 Goal 按依赖关系拆分为多个 Project 和 Task，节点可拖拽、
键盘快捷操作（类思维导图），连线即依赖，整张图是一个 DAG。

**技术选型**：纯 vanilla JS + SVG，零运行时依赖。开发时拆模块 + Vitest
单测，由极简拼装脚本内联为最终单 HTML 文件交付。

## 数据模型

```
Goal
├── title: 目标名称（根节点显示）
├── nodes: [{ id, title, description, status(todo/doing/done),
│             estimatedHours, deadline, type(project/task),
│             x, y(手动拖拽覆盖位置), collapsed(前序折叠),
│             detailOpen(详情面板展开) }]
└── edges: [{ from, to }]   // from 完成后才能做 to
```

- **两类节点**：`project`（一级节点，直连 Goal 根，视觉醒目）和 `task`。
  结构相同。
- **边即依赖**：Tab/Enter 创建节点时自动建边；可手动拖线形成多前序（DAG）。
  删除节点/边时清理悬空引用。
- **环检测**：连线成环时拒绝并提示。
- **前序步骤不存储**：从 edges 实时反推。
- **可开始高亮**：status 为 todo 且所有前序均 done 的节点自动发光边框。
- **位置**：默认自动布局；手动拖过的节点记录 x/y 覆盖；
  "整理布局"按钮清除所有覆盖。
- **持久化**：整个 Goal 序列化 JSON 存 localStorage，操作即时保存；
  支持导出/导入 JSON 文件。

## 层级结构

一个页面 = 一个 Goal。Goal 为画布最左侧根节点，Project 为一级节点，
Task 在 Project 下游向右展开。同一画布，支持跨 project 连依赖。

## 交互设计

### 键盘（选中节点时）

| 键 | 行为 |
|---|---|
| `Tab` | 创建后继任务（右侧，自动连边 当前→新），立即编辑 |
| `Enter` | 创建并行节点（继承当前节点的所有前序），立即编辑 |
| 双击 / `F2` | 编辑标题，`Enter` 确认，`Esc` 取消 |
| `Delete`/`Backspace` | 删除节点及相关边（二次确认） |
| 方向键 | 沿连线移动选中 |
| `Space` | 状态循环 todo → doing → done |
| `D` | 切换详情面板 |
| `Esc` | 取消选中 |

输入框聚焦时所有画布快捷键失效。

### 鼠标

- 拖节点 = 移动（记手动覆盖）；从节点右侧连接点拖线 = 建依赖
- 右键点边 = 删除依赖；空白拖拽 = 平移；滚轮 = 缩放（25%–200%）
- 双击空白 = 创建游离任务
- 新建节点在视野外时自动滚动跟随

### 节点的两个独立展开/收敛操作

1. **前序步骤折叠**（节点左侧 `▸/▾`）：收起所有汇入此节点的上游前序链，
   左侧显示折叠数角标（如 `◂5`）。若某前序同时连向其他可见节点则不隐藏
   （避免断图）。类似思维导图分支折叠，但方向是上游。
2. **任务详情面板**（`⊕` 图标或 `D` 键）：节点下方展开面板——描述（多行
   可编辑）、状态下拉、预计工时、截止日期。前序列表也在此显示，每条带
   状态色点，点击跳转选中。两个操作互不影响，可同时多节点展开。

## 自动布局与视觉

- **分层布局**（简化 Sugiyama）：层级 = max(前序层级) + 1，层从左到右，
  同层垂直排列，排序靠近前序平均纵坐标以减少交叉。
- 连线为 SVG 三次贝塞尔曲线（水平出入）带箭头。
- 手动拖过的节点不参与重算；"整理布局"全部重排（有确认）。
- 折叠/详情展开时其余节点平滑让位（CSS transition）。
- **风格**：浅色画布 + 点阵网格；节点圆角卡片，状态色 灰(todo)/蓝(doing)/
  绿(done)/金色发光(可开始)；截止日期过期未完成显示红色警示角标；
  Project 节点更大加粗带主题色边框，Goal 根最醒目。
- **工具栏**：Goal 标题（可编辑）、整理布局、导出/导入 JSON、缩放显示/重置。

## 边界情况

- 循环依赖：拒绝连接，目标节点闪红。
- 删除有下游的节点：确认框提示断开 N 条依赖，下游节点保留。
- localStorage 不可用/写满：黄条警告"请手动导出"。
- 导入非法 JSON：schema 校验失败则提示且不覆盖。
- 导入前自动备份旧数据到备份槽，提供"撤销导入"。

## 测试方案

- `core/` 纯函数模块（数据操作、环检测、布局、前序推导、可开始判定、
  JSON 校验）：Vitest 单测，TDD 先行，覆盖率 80%+。
- `ui/` DOM 渲染与事件薄层。
- 拼装脚本把模块内联为最终单 HTML。
- Playwright E2E：建节点 → Tab/Enter → 拖线 → 折叠 → 刷新持久化 → 导出导入。
